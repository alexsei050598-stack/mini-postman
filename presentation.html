Mini Postman

Легкий веб-клиент для API-тестирования с фокусом на QA-сценарии.

HTTP methods Network modes Error resilience

Цель: показать не просто "fetch-кнопку", а инструмент, который помогает быстро находить проблемы в API.

Проблема

Почему не хватало обычного mini client

  • Только GET/POST и слабая диагностика ошибок.
  • Нет истории и повторного запуска запросов.
  • Пустые ответы (HEAD/204) часто ломают парсинг.
  • Сложно воспроизвести сетевые кейсы для QA.
Решение

Что реализовано

  • Методы: GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS.
  • Headers + body (с валидацией JSON).
  • Статус-бейджи: 2xx, 4xx, 5xx.
  • История запросов в localStorage.
  • Режимы: Normal, Slow 3G, Offline.
  • Copy response, Clear, подсказки по ошибкам.
  • Корректная обработка no-content ответов.
  • Текст/JSON/бинарный fallback без падений.
Архитектура

Просто и прозрачно

  1. UI: `index.html` + `style.css`.
  2. Логика: `script.js` (валидация, fetch, рендер).
  3. Хранилище: localStorage для истории.
  4. Поток: форма -> валидация -> запрос -> анализ -> сохранение.
Демо

Сценарий показа (2-3 минуты)

  1. GET `.../posts/1` -> 200 + JSON.
  2. POST `.../posts` -> 201 + тело ответа.
  3. HEAD `.../posts/1` -> 200 + `[no content]`.
  4. 404 endpoint -> подсказка про неверный путь.
  5. Slow 3G и Offline -> проверка устойчивости.
  6. Клик по истории -> авто-подстановка формы.
Ценность

Что видит работодатель

  • Ты делаешь не только "работает", но и "удобно тестировать".
  • Есть QA-мышление: edge cases, ясные ошибки, воспроизводимость.
  • Есть продуктовый подход: UX, демо-готовность, аккуратная подача.

Этот проект уже выглядит как практичный инструмент, а не учебный черновик.

Финал

Короткий спич

Mini Postman — это QA-ориентированный API-клиент, который поддерживает основные HTTP-методы, историю, режимы сети и корректную обработку сложных ответов.

Инструмент помогает быстро локализовать проблемы в запросе, сети и сервере, а интерфейс позволяет использовать его как демо-продукт для портфолио.
