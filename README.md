# Mini Postman (QA demo)

Лёгкий web-инструмент для тестирования API прямо из браузера: запросы, хедеры, история, подсветка статусов и QA-режимы.

## Возможности
- GET/POST запросы с кастомными headers и телом (JSON валидация перед отправкой).
- Подсветка статусов: бейджи 2xx / 4xx / 5xx.
- Быстрая аналитика: подсказки для 404/403/5xx.
- История (localStorage) с автозаполнением по клику.
- QA-режимы: Normal, Slow 3G (эмуляция задержки), Offline (форсируем ошибку сети).
- Copy response, Clear форма/ответ.
- Красивый UI в стиле банковского приложения.

## Локальный запуск
```bash
# просто открыть файл
open index.html
# или через любой статический сервер
python3 -m http.server 3000
```

## Деплой
### GitHub Pages
1. Закоммитьте репозиторий.
2. В GitHub Settings → Pages выберите ветку `main` (или `gh-pages`) и корень `/`.
3. Через пару минут приложение будет доступно по `https://<user>.github.io/<repo>/`.

### Vercel
1. Создайте новый проект, укажите этот репозиторий.
2. Framework = **Other**, build-команда не требуется, output — корень (`/`).
3. Deploy. Vercel выдаст прод-URL вида `https://<project>.vercel.app`.

## Структура
- `index.html` — разметка и подключение скриптов/стилей.
- `style.css` — оформление (градиенты, карточки, бейджи).
- `script.js` — логика запросов, история, симуляция сети, подсказки QA.

## Подсказки по использованию
- Для проверки ошибок: `https://httpbin.org/status/404`, `.../500`.
- Для эхо POST: `https://httpbin.org/post`.
- История сохраняется локально; очистка кэша браузера её сбросит.
